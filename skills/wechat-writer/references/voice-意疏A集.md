## 共性观察：意疏 voice 的真实底色（基于 9 篇）

### ✅ 该保留的（教 AI 模仿）

1. **保姆级步骤感**
   "我们点击 X" → "我们点击 Y"，每步配截图 `[IMG]`。
   降低读者认知负担，新手友好。教程类必备。

2. **章节标题用 emoji 锚点**（🟧/🍬/💡/✅/🚀）
   每个一级章节配一个 emoji，作为视觉路标。

3. **品牌化签名结尾**
   "意气风发，漫卷疏狂"+"我叫意疏，希望我们一起成长。下次见！"
   9/9 篇都有，是品牌资产。详见 `references/意疏-签名结尾.md`。

4. **第 #3 篇出现过的高质量个人句**：
   > 访问量会到百万，但我会先成为那个值得被看见的人。
   这种句子的密度需要提升 —— 这才是意疏 voice 的瞬间。

### ⚠️ 该升级的（教 AI 不要再这样写）

1. **"想象一下，你..."的钩子用了 4/9 篇** —— 太模板。换成"昨天/今天/上周我..."的具体场景开头。
2. **"三句话总结核心价值"+营销破折号长句** —— 是商业征文模板，不是日常 voice。
3. **"💬欢迎交流 / 👍点赞、收藏" 引流前缀** —— CSDN 平台习惯，公众号绝对不要带。
4. **段落长度均匀** —— 缺"短-长-短"呼吸。每段控制在 1-3 句，刻意打破均匀。
5. **个人态度密度偏低** —— 自嘲 / 反讽 / 情绪化口语几乎没有。需要刻意补。

## A 集 · 教程类范文（3 篇，可放心仿写结构）

### A1《小龙虾一篇讲透，从零到跑起来》OpenClaw 部署
- 日期：2026-04-29  字数：3195
- **结构**：一、部署 → 二、实操讲解 → 三、定时任务 → 总结 → 签名
- **学什么**：
  - 步骤拆得极细，每步一句话 + 一张截图占位
  - "tips：虽然现在腾讯云可以一键扫码……但我还是推荐"——给读者**做选择题**，不是抽象指引
  - 总结段："你不再只是考虑'能用'，而开始思考怎么长期用时，OpenClaw 才会慢慢显出价值"——**升华到使用哲学**而不是堆功能

### A8《超干货！手把手教你如何在本地部署 DeepSeek》
- 日期：2025-03-19  字数：2046
- **结构**：前言 → 安装 ollama → 拉取模型 → 接入 chatbox → 结语
- **学什么**：
  - 平均每 200 字一张截图占位 —— 读者不会迷路
  - "注：这里我们的命令行界面不要关闭"—— **预警式注释**，主动避坑
  - 标题"超干货！" "手把手"+"快速掌握"+"高效上手"四个利益词堆叠 —— 教程类标题套路（可学，但别每篇都用）

### A9《C 语言新手小白详细教程（2）》
- 日期：2024-07-28  字数：6830
- **学什么**：
  - 开头"序（一些闲话）"是教程类**最有 voice 的写法**：
    > 我希望我的语言不要像专业书那样让人眼花缭乱，所以当我解释语法时，我会尽量避免使用太多专业术语。
  - 这种"我希望……所以我尽量……"的承诺式开场，比"想象一下"高一个等级
  - 章节用阅读指南做目录 —— 长文（6000+ 字）必备

### A 集仿写公式

```
开头 = 序/前言（说一句"我希望……"的承诺，或承认踩过的坑）
   ↓
章节 1 = emoji + 标题 → 步骤 1-N（每步一张图）
   ↓
章节 2 = emoji + 标题 → ...
   ↓
结语 = 升华到"长期使用"或"使用哲学"，不是功能复述
   ↓
签名 = 调用 references/意疏-签名结尾.md
```

### B3《豆包 Doubao-Seed-Code 深度测评：直接换掉 Claude Code！》
- 日期：2025-11-24  字数：4104
- **唯一进 B 集的篇，因为有这一句**：
  > 访问量会到百万，但我会先成为那个值得被看见的人。
- **学什么**：
  - 开头先表态自己有什么"主力工具"+"短板列表"，再引出新主角 —— **建立对比基线**

### ❌ B 集集体避雷清单（来自 #2 #4 #5 #6 商业征文）

| 模板 | 例句 | 为什么不要 |
| --- | --- | --- |
| "想象一下，你..." 钩子 | "想象一下，你需要设计系列宣传图..." | 4 篇都用过了，AI 一学就成了你的标签，立刻识别出"AI写的" |
| "三句话总结核心价值" 模板 | "1. '解析不卡壳，还能变资产'——针对..." | 商业稿专属，公众号写这个会很硬 |
| "💡技术的价值不在于'能做什么'..." 引言块 | 章节起首一段引号包名言 | 端着、说教、像广告 |
| 长段无换行 | 单段 200+ 字密集排比 | 公众号读者只看手机，3 行换一段是底线 |
| 厂商定位句 | "TextIn 功能矩阵完善，涵盖..." | 像产品页文案，不像评测 |

### B 集仿写公式（升级后的，对标卡兹克）

```
开头 = 一个具体场景/时间戳（"昨天我..."），不是"想象一下"
   ↓
我的现状/对比基线（我用什么、它的短板）
   ↓
新主角登场（一句话定位 + 一句话怀疑）
   ↓
实测分章（带情绪：哪里被惊艳、哪里翻车）
   ↓
"以前 vs 现在" 收尾
   ↓
签名
```

## 写作前自检清单

下笔前对照这 5 条：

- [ ] 第一句不是"想象一下"——而是具体时间/场景
- [ ] 全文至少 3 处个人态度词（"我有点麻了""我心里咯噔一下""我浑身难受"这种）
- [ ] 没有"💬欢迎交流" "👍点赞收藏" 这类引流前缀
- [ ] 段落短-长-短交替，不是 8 段都 50 字
- [ ] 结尾不复述功能，给一句"使用哲学"或"反转感受"
- [ ] 签名段调用 `references/意疏-签名结尾.md`，不要让 AI 自由发挥结尾
