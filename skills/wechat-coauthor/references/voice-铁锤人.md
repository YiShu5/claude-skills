# voice-铁锤人 · 辅助参照（教程/技术评测类）

> **使用场景**：写 AI 工具教程、技术评测、新产品介绍时，读这份文档。
> 个人故事 / 情感叙事 → 看 `voice-黄叔.md`。
> **禁止**：借整体 voice 风格。铁锤人的技法可以借，他的语气不借——他的语气是他的，意疏的语气是意疏的。

---

## 铁锤人的 3 个可借技法

### 技法 1：把技术行为拟人化

不用技术术语描述 agent 行为，用八卦口吻描述"人物行为"。

**原句**：「凯西不老实，跑去抢别人的活了」

**句式**：`[角色名] + 口语评价（不老实/偷懒/出轨）+ 描述行为`

**为什么有力**：
- "agent 出现了越界行为" = 读者需要先懂 agent 是什么
- "凯西不老实，跑去抢别人的活了" = 所有人秒懂

**意疏怎么用**（写 AI 工具行为时）：
- ❌ "模型在推理过程中出现了超出预期的输出"
- ✅ "它干完自己的活，盯了隔壁一眼，跑过去把那份也做了"
- ✅ "任务完成了，它没退出，在那儿等着——像个新来的实习生不知道下班"

**什么时候用**：描述 AI/工具做了一个"出乎意料"的行为时。

---

### 技法 2：先替读者说出疑虑，再打脸

在最震撼的案例出现前，先替读者说一句"这有什么用"。

**原句**：「但你可能在想，看起来没什么用吧？」（然后接 11 小时 73 篇论文）

**句式**：`但你可能在想，[最大疑虑]？` → `[直接给最强证据，不解释]`

**为什么有力**：
- 读者脑子里有疑虑 = 读者在准备关掉页面
- 先说出来 = 读者觉得"哦你懂我"，留下来听
- 然后直接打脸，不解释 = 让证据自己说话

**意疏怎么用**：
- "你可能觉得这玩意没什么用。" → 然后贴结果截图，不加解释
- "听起来很麻烦对不对。" → 然后："其实就一条命令。"

---

### 技法 3：用对比公式定义新概念

把一个新东西和已知的东西放在对比公式里，读者不需要理解新概念就能定位它。

**原句**：「agent 框架是给 AI 当老板的，WorldSeed 是给 AI 当上帝的」

**句式**：`[已知事物] 是给 X 当 [角色A] 的，[新事物] 是给 X 当 [角色B] 的`

**意疏怎么用**（介绍新工具时）：
- "Claude.ai 是让 AI 帮你干活的，Claude Code 是让 AI 替你干活的"
- "wechat-formatter 是 AI 的排版工，wechat-publisher 是 AI 的快递员"

## 铁锤人 voice 的特征（学结构，不学语气）

铁锤人的文章结构是可以借鉴的，但他的**语气不要借**：

| 铁锤人的语气 | 原因不借 |
|---|---|
| "废话少说" | 是他的口头禅，意疏里用显得刻意 |
| 大量感叹号 | 铁锤人的节奏，意疏更克制 |
| "你最想创造一个什么样的世界" 式升华 | 意疏结尾不升华，落画面 |

**铁锤人的结构可以借**：
- "是什么 → 怎么装 → 能玩什么" 三段式（教程类实用）
- 每个大案例前先讲"最离谱的一次"（先给结论，再给过程）
- 代码块和操作命令单独一行，不埋在段落里
